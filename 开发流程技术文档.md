## 1. 项目背景与目标

### 1.1 项目概述

本项目旨在构建一个基于 **LangGraph + ChromaDB** 的智能 Agent 系统，实现 RAG（检索增强生成）能力的私有化知识库问答应用。系统支持多轮对话、语义检索、工具调用等核心能力，可本地部署运行，无需依赖云服务。

### 1.2 核心目标

| 目标 | 描述 |
|------|------|
| 智能对话 | 基于大语言模型的多轮对话，支持上下文记忆 |
| 知识检索 | 基于 ChromaDB 向量数据库的语义搜索 |
| 工具调用 | Agent 自主决策调用计算器、时间查询等工具 |
| 私有部署 | 所有组件本地运行，数据不出本机 |
| 多模型适配 | 通过 LiteLLM 支持 100+ LLM 模型切换 |

### 1.3 目标用户

- AI 工程师：构建和部署智能 Agent 系统
- 研究人员：进行大模型应用研究和实验
- 企业用户：搭建私有知识库问答系统

### 3.4 LangGraph 图结构

```python
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.prebuilt import ToolNode, tools_condition

graph = StateGraph(MessagesState)

# 添加节点
graph.add_node("agent", agent_node)      # LLM 推理节点
graph.add_node("tools", ToolNode(tools)) # 工具执行节点

# 定义边
graph.add_edge(START, "agent")                         # 入口 → Agent
graph.add_conditional_edges("agent", tools_condition)  # Agent → 条件路由
graph.add_edge("tools", "agent")                       # Tools → Agent（循环）

# 编译图（带持久化）
from langgraph.checkpoint.memory import MemorySaver
checkpointer = MemorySaver()
app = graph.compile(checkpointer=checkpointer)
```

### 5.3 阶段三：知识库构建模块

#### 5.3.1 文本分块策略

采用基于标点符号的智能分块，保证语义完整性：

```python
def chunk_text(text: str, chunk_size: int = 500, overlap: int = 50) -> list[str]:
    """智能文本分块：在句号、问号、感叹号等处切分"""
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        if end < len(text):
            # 在分块末尾附近查找最佳切分点
            for sep in ['。', '！', '？', '.', '!', '?', '\n']:
                pos = text.rfind(sep, start + chunk_size - overlap, end)
                if pos != -1:
                    end = pos + 1
                    break
        chunks.append(text[start:end])
        start = end - overlap  # 保留重叠部分确保上下文连续
    return chunks
```

#### 5.3.2 知识导入流程

```python
def import_knowledge(title: str, content: str, source: str = "manual"):
    # 1. 文本分块
    chunks = chunk_text(content, settings.CHUNK_SIZE, settings.CHUNK_OVERLAP)

    # 2. 写入 SQLite 元数据
    doc_id = generate_doc_id()
    save_document_meta(doc_id, title, source, len(chunks))

    # 3. 批量写入 ChromaDB 向量库
    records = [
        {
            "doc_id": doc_id,
            "chunk_id": f"{doc_id}_{i}",
            "title": title,
            "content": chunk,
            "metadata": json.dumps({"chunk_index": i, "source": source})
        }
        for i, chunk in enumerate(chunks)
    ]
    vector_db.batch_insert("knowledge_chunks", records)

    return doc_id, len(chunks)
```

### 5.4 阶段四：Agent 工具定义

使用 LangChain `@tool` 装饰器定义工具，配合 Pydantic Schema 描述参数：

```python
from langchain_core.tools import tool
from pydantic import BaseModel, Field

class SearchInput(BaseModel):
    query: str = Field(description="搜索查询文本")
    top_k: int = Field(default=5, description="返回结果数量")

@tool(args_schema=SearchInput)
def search_knowledge_base(query: str, top_k: int = 5) -> str:
    """在知识库中进行语义搜索，返回与查询最相关的知识片段。"""
    results = vector_db.search(
        table_name="knowledge_chunks",
        query_text=query,
        limit=top_k
    )
    return format_search_results(results)

@tool
def get_current_time() -> str:
    """获取当前日期和时间。"""
    from datetime import datetime
    return f"当前时间：{datetime.now().strftime('%Y-%m-%d %H:%M:%S')}"

@tool
def calculate(expression: str) -> str:
    """执行数学计算表达式。"""
    # 安全计算：仅允许数字和基本运算符
    allowed = set("0123456789+-*/().% ")
    if not all(c in allowed for c in expression):
        return "错误：表达式包含非法字符"
    return f"计算结果：{eval(expression)}"

# 工具列表（供 ToolNode 使用）
ALL_TOOLS = [search_knowledge_base, get_current_time, calculate]
```

#### 5.5.1 系统 Prompt 设计

```python
SYSTEM_PROMPT = """你是一个智能知识库助手。你可以回答用户问题，也可以使用工具来获取信息。
```

#### 5.5.2 Agent 节点实现

```python
from langchain_core.messages import SystemMessage, HumanMessage, AIMessage

def agent_node(state: dict) -> dict:
    """Agent 推理节点：调用 LLM 并处理工具调用"""
    messages = state["messages"]

    # 构造带系统提示的消息列表
    full_messages = [SystemMessage(content=SYSTEM_PROMPT)] + messages

    # 通过 LiteLLM 调用 LLM
    from litellm import completion
    response = completion(
        model=f"{settings.LLM_PROVIDER}/{settings.LLM_MODEL}",
        messages=[m.dict() for m in full_messages],
        tools=[t.get_function_schema() for t in ALL_TOOLS],
        temperature=settings.LLM_TEMPERATURE,
        max_tokens=settings.LLM_MAX_TOKENS,
        api_key=settings.LLM_API_KEY,
        api_base=settings.LLM_API_BASE,
    )

    ai_message = response.choices[0].message
    return {"messages": [ai_message]}
```

#### 5.5.3 图构建与编译

```python
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.prebuilt import ToolNode, tools_condition
from langgraph.checkpoint.memory import MemorySaver

def build_agent_graph():
    """构建 LangGraph Agent 工作流"""
    builder = StateGraph(MessagesState)

    # 添加节点
    builder.add_node("agent", agent_node)
    builder.add_node("tools", ToolNode(ALL_TOOLS))

    # 定义执行流
    builder.add_edge(START, "agent")
    builder.add_conditional_edges("agent", tools_condition)
    builder.add_edge("tools", "agent")

    # 编译并添加持久化
    checkpointer = MemorySaver()
    return builder.compile(checkpointer=checkpointer)

# 实例化
agent = build_agent_graph()
```

### 5.6 阶段六：对话服务层

```python
import uuid

class ChatService:
    def __init__(self):
        self.agent = build_agent_graph()

    def chat(self, message: str, thread_id: str = None, user_id: str = "default") -> str:
        """同步对话接口"""
        if not thread_id:
            thread_id = str(uuid.uuid4())

        config = {"configurable": {"thread_id": thread_id}}

        # 调用 Agent
        result = self.agent.invoke(
            {"messages": [HumanMessage(content=message)]},
            config=config
        )

        # 持久化到 SQLite
        save_message(thread_id, user_id, "user", message)
        ai_response = result["messages"][-1].content
        save_message(thread_id, user_id, "assistant", ai_response)

        return ai_response, thread_id

    def chat_stream(self, message: str, thread_id: str = None):
        """SSE 流式对话接口"""
        if not thread_id:
            thread_id = str(uuid.uuid4())

        config = {"configurable": {"thread_id": thread_id}}

        for chunk in self.agent.stream(
            {"messages": [HumanMessage(content=message)]},
            config=config
        ):
            if "agent" in chunk:
                yield chunk["agent"]["messages"][-1]
```

## 6. 关键技术点解析

### 6.1 LangGraph 核心概念

| 概念 | 说明 | 本项目应用 |
|------|------|-----------|
| **Node（节点）** | 编码 Agent 逻辑的函数 | `agent_node`（LLM 推理）、`ToolNode`（工具执行） |
| **Edge（边）** | 决定下一个执行的节点 | 固定边 + 条件边（`tools_condition`） |
| **State（状态）** | 节点间传递的共享数据 | `MessagesState`（消息列表） |
| **Super-step** | 并行执行模型 | Agent 与 Tools 的交替执行 |
| **Checkpointer** | 状态持久化 | `MemorySaver` 保存对话上下文 |

### 6.2 LiteLLM 多模型适配

LiteLLM 提供统一的 LLM 调用接口，支持 100+ 模型无缝切换：

### 6.3 RAG 检索增强生成


| LLM 调用失败 | API Key 无效 | 检查 `.env` 中 `LLM_API_KEY` |
| 对话无上下文 | `thread_id` 不一致 | 确保同一会话使用相同 `thread_id` |
| 向量搜索无结果 | 知识库为空 | 先通过导入接口添加知识 |
| 内存占用过高 | 历史消息过多 | 调整 `MAX_HISTORY_MESSAGES` |
| Docker 启动失败 | 端口冲突 | 修改 `docker-compose.yml` 端口映射 |
