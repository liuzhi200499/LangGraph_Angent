# LangGraph — 错误处理与重试策略

---

## 节点级别的错误处理

节点内的错误处理有一个核心原则：**尽量不要让异常从节点中抛出**。如果一个节点抛出未捕获的异常，整个图的执行会立即终止，用户会看到一个错误信息，之前的所有工作都白费了。

更好的做法是在节点内部 try-except 捕获异常，然后返回一条友好的错误消息（比如 AIMessage），让图继续运行。这样用户至少能得到一个有意义的回复，而不是一个崩溃信息。

```python
def safe_agent_node(state: MessagesState) -> dict:
    """带错误处理的 Agent 节点"""
    try:
        messages = state["messages"]
        response = call_llm(messages)

        # 检查 LLM 返回是否有效
        if not response:
            return {"messages": [AIMessage(content="抱歉，我暂时无法回答。")]}

        return {"messages": [response]}

    except Exception as e:
        # 不抛异常，而是返回错误消息让图继续运行
        error_msg = f"Agent 处理出错：{type(e).__name__}"
        return {"messages": [AIMessage(content=error_msg)]}
```

## 工具级别的错误处理

工具函数的错误处理比节点更关键，因为工具直接面向外部系统（数据库、API、文件系统），出错概率更高。和节点一样，工具函数应该捕获异常并返回错误描述，而不是抛异常。这样即使某个工具调用失败，Agent 还能看到错误信息，决定是重试、换一个工具、还是直接告诉用户。

```python
@tool
def search_knowledge(query: str) -> str:
    """搜索知识库（带错误处理）"""
    try:
        results = vector_db.search(query)
        if not results:
            return "未找到相关内容。"
        return format_results(results)
    except ConnectionError:
        return "知识库连接失败，请稍后重试。"
    except Exception as e:
        return f"搜索出错：{e}"
```

## 带重试的 LLM 调用

LLM API 调用是最容易出错的环节之一。常见的错误包括限流（Rate Limit Error）、连接超时、服务端临时不可用等。对于这些瞬时错误，重试是最简单有效的应对策略。

重试策略的关键是"指数退避"：每次重试前等待的时间翻倍（1秒、2秒、4秒）。这样既能给 API 服务恢复的时间，又不会对服务端造成持续压力。一般 3 次重试就足够了——如果连续 3 次都失败，很可能是更严重的问题，应该向上抛出异常。

```python
import time

def call_llm_with_retry(messages, tools=None, max_retries=3):
    """带重试和退避的 LLM 调用"""
    for attempt in range(max_retries):
        try:
            return completion(
                model="deepseek/deepseek-chat",
                messages=messages,
                tools=tools,
                api_key="your-key"
            )
        except RateLimitError:
            wait = 2 ** attempt  # 1s, 2s, 4s
            print(f"触发限流，等待 {wait}s 后重试...")
            time.sleep(wait)
        except APIConnectionError:
            if attempt < max_retries - 1:
                time.sleep(1)
            else:
                raise
    raise Exception(f"LLM 调用失败，已重试 {max_retries} 次")
```
