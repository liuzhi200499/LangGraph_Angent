# LangGraph — v1.x 新特性速览

---

## Functional API（函数式 API）

LangGraph v1 新增了函数式 API，无需手动构建 StateGraph，用装饰器即可定义工作流：

```python
from langgraph.func import entrypoint, task
from langgraph.checkpoint.memory import InMemorySaver

@task
def search_task(query: str) -> str:
    """定义一个任务（等价于 Node）"""
    return f"搜索结果：{query}"

@entrypoint(checkpointer=InMemorySaver())
def my_workflow(inputs: dict, *, previous=None):
    """定义工作流入口（等价于 Graph）"""
    result = search_task(inputs["query"]).result()
    return {"answer": result}

# 调用方式与 Graph API 一致
config = {"configurable": {"thread_id": "1"}}
result = my_workflow.invoke({"query": "Python"}, config=config)
```

**对比 Graph API vs Functional API：**

| 维度 | Graph API（本文档主要讲解） | Functional API |
|------|---------------------------|----------------|
| 定义方式 | `StateGraph` + `add_node` + `add_edge` | `@task` + `@entrypoint` 装饰器 |
| 适用场景 | 复杂的图结构、条件路由、多分支 | 线性/简单工作流、快速原型 |
| 状态管理 | 自定义 TypedDict | 自动管理 |
| 学习曲线 | 较陡 | 较平缓 |

## Command 类型：节点内动态路由

v1 新增 `Command` 类型，节点可以直接在返回值中指定下一个要执行的节点，无需单独定义条件边：

```python
from langgraph.types import Command

def agent_node(state: MessagesState):
    response = call_llm(state["messages"])

    if response.tool_calls:
        # 动态指定下一个节点，替代 add_conditional_edges
        return Command(
            update={"messages": [response]},
            goto="tools"
        )
    return {"messages": [response]}

# 使用 Command 后，无需再写 tools_condition 路由函数
builder.add_node("agent", agent_node)
builder.add_node("tools", ToolNode(ALL_TOOLS))
builder.add_edge(START, "agent")
builder.add_edge("tools", "agent")
# 注意：agent 节点的 goto 会自动处理路由
```

## Long-term Memory（跨会话记忆）

Checkpointer 提供**短期记忆**（同一 thread_id 内），而 `BaseStore` 提供跨 thread_id 的**长期记忆**：

```python
from langgraph.store.memory import InMemoryStore

store = InMemoryStore()

# 跨会话存储用户偏好
store.put(("user", "user_001"), "preferences", {"language": "zh", "topics": ["AI"]})

# 在任何 thread_id 的对话中读取
prefs = store.get(("user", "user_001"), "preferences")
```

| 记忆类型 | 实现方式 | 作用域 | 生命周期 |
|----------|----------|--------|----------|
| 短期记忆 | `InMemorySaver` / Checkpointer | 同一 `thread_id` | 单次会话 |
| 长期记忆 | `InMemoryStore` / 外部 Store | 跨 `thread_id` | 永久（直到手动删除） |

## create_react_agent → create_agent 迁移

LangGraph v1 中，`langgraph.prebuilt.create_react_agent` 已弃用，推荐使用 LangChain 新的 `create_agent` API：

```python
# 旧方式（已弃用）
from langgraph.prebuilt import create_react_agent
agent = create_react_agent(model, tools)

# 新方式（v1 推荐）
from langchain.agents import create_agent
agent = create_agent(model, tools)
```

> 手动构建 StateGraph（本文档讲解的方式）仍然完全有效且推荐用于复杂场景。
> `create_agent` 更适合快速搭建标准 ReAct Agent。
